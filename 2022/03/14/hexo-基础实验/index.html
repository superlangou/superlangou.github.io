linux小游戏

发表于2022-03-14更新于2022-03-20

前期准备

1.下载安装包 链接：https://www.cs.cmu.edu/~213/bootcamps/lab1_handout.tar
2.将安装包 粘贴进虚拟机。
3.右键 在终端运行
4.输入./bomb 开始闯关

第一关

大致意思是你需要阅读/university_center目录下stones.txt，并解救他。
进入文件目录
输入cd 指令 进入university_center目录
输入 cat stones.txt查看文本。
文本中的大致意思是 需要你将stone.txt连同他的父目录university_center 一起移动到free_the目录下，并使用pwd指令所得到的地址及为第一关答案
使用 cd 移动目录 目的目录 移动即可。
然后到达移动后的stone.txt的所在目录下，输入pwd，即可得到第一关答案.

第二关

大致意思是 你需要保护任何stones 不被关在 capture_the目录下
使用 rmdir capture_the 命令 删除 capture_the目录即可.

第三关

大致意思是我们需要在 evil plan目录下的文件中，找到包含 “secret message:” 的文本文件。
使用 grep ‘secret message:’ * 查找即可找到密码。(注意搜索关键词一定要用单引号圈起来！)

第四关

大致意思是问你需要什么命令来查看 evil.txt文件
查看文本文件使用cat命令即可.

第五关

大致意思是让你打开evil24.txt去寻找信息,打开后发现是一群排序混乱且重复的语句，需要我们进行排序和删除。
使用 sort -n evil24.txt 进行排序。
再使用 uniq evil24.txt 删除重复行 即可查看信息。
处理后的信息大致意思为在we_tried_to目录下创建一个名为 release的文件。
所以答案为 touch release.txt

第六关

大致意思是信息在start/we_tried_to/them/one目录下的listened.c文件中，但缺少了某些东西.
我们打开目标文件，文件中提示你缺少了一部分命令在no.txt文本中，我们找到文本并将丢失命令粘贴回文件即可。
然后我们根据提示 输入 gcc listened.c 进行编译文件（需要提前下载gcc插件）
然后输入 ./a.out 运行文件
提示你第一行的数字即为答案。
使用sort命令进行排序即可找到答案

第七关

大致意思为 你需要删除 a.out文件。然后密码就藏在所有目录的目录名中.
我们使用tree命令查看文件树(需要提前下载tree插件)
根据文件数输入答案即可（注意将下划线改为空格）
